1. Etiquetas Estructurales

Estas etiquetas forman el esqueleto de una página web. Le dicen al navegador cómo está organizado el documento.

<!DOCTYPE html>

¿Qué es? Nos dice en que archivo vamos a trabajar

Sintaxis:

<!DOCTYPE html>
Buenas prácticas: Siempre se escribe en la primera línea. Sin ella el navegador puede mostrar la página de forma incorrecta.

<html>

¿Qué es? Es el idioma de todo el contenido de la página.

Sintaxis:

<html lang="es">
  ... todo el contenido va aquí ...
</html>
Buenas prácticas: Siempre agrega el atributo lang="es" para indicar el idioma.

<head>

¿Qué es? Contiene información sobre la página que el usuario NO ve: título, estilos, metadatos.

Sintaxis:

<head>
  <meta charset="UTF-8">
  <title>Mi página</title>
</head>
Buenas prácticas: Siempre incluye <meta charset="UTF-8"> para que admitan las tildes y la ñ.

<body>

¿Qué es? Aquí va TODO el contenido visible de la página: textos, imágenes, botones, etc.

Sintaxis:

<body>
  <h1>Hola mundo</h1>
  <p>Este es mi primer párrafo.</p>
</body>
Buenas prácticas: Todo lo que quieras que el usuario vea debe estar dentro de <body>.

2. Etiquetas de Texto

Estas etiquetas dan formato y significado al texto de la página.

<h1> hasta <h6> — Títulos

¿Qué es? Son los títulos de la página. h1 es el más grande e importante, h6 el más pequeño.

Sintaxis:

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Demostración:

Título h1

Título h2

Título h3

Buenas prácticas: Usa solo un <h1> por página. No saltes niveles (no vayas de h1 a h3 sin h2). JERARQUIA MUSHASO

<p> — Párrafo

¿Qué es? Define un párrafo de texto. El navegador deja un espacio antes y después automáticamente.

Sintaxis:

<p>Este es un párrafo de ejemplo.</p>

Demostración:

Este es un párrafo de ejemplo con texto normal.

Buenas prácticas: No uses <br> seguidos para crear espacio. Para eso usa un párrafo nuevo.

<strong> y <em> — Negrita y cursiva

¿Qué es? <strong> pone el texto en negrita, <em> pone el texto en cursiva con énfasis.

Sintaxis:

<p>Este texto es <strong>muy importante</strong>
y este es <em>especial</em>.</p>

Demostración:

Este texto es muy importante y este es especial.

Buenas prácticas: Usa <strong> y <em> en vez de <b> e <i>. Los primeros tienen significado semántico.

<a> — Enlace

¿Qué es? Crea un enlace a otra página o sitio web. El atributo href indica a dónde va el enlace.

Sintaxis:

<a href="https://youtube.com/@carlosjuliocadena3741?si=H-8ZJAMd1zTrp47d">Ir a Google</a>

<!-- Para abrir en pestaña nueva: -->
<a href="https://youtube.com/@carlosjuliocadena3741?si=H-8ZJAMd1zTrp47d" target="_blank">Abrir en nueva pestaña</a>

Demostración:

Ir a CalicheSoft
Buenas prácticas: El texto del enlace debe describir a dónde lleva. Evita escribir "haz clic aquí".

3. Etiquetas de Listas

Las listas sirven para organizar información en puntos o pasos.

<ul> — Lista sin orden (puntos)

¿Qué es? Crea una lista con viñetas (puntos). Se usa cuando el orden no importa. Cada elemento va dentro de <li>.

Sintaxis:

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Demostración:

  • HTML
  • CSS
  • JavaScript
Buenas prácticas: Úsala para listas donde el orden no importa, como una lista de ingredientes o características.

<ol> — Lista ordenada (números)

¿Qué es? Crea una lista numerada. Se usa cuando el orden sí importa, como pasos de un proceso.

Sintaxis:

<ol>
  <li>Abrir el editor de código</li>
  <li>Escribir el código HTML</li>
  <li>Guardar el archivo</li>
  <li>Abrir en el navegador</li>
</ol>

Demostración:

  1. Abrir el editor de código
  2. Escribir el código HTML
  3. Guardar el archivo
  4. Abrir en el navegador
Buenas prácticas: Úsala para instrucciones, recetas o pasos donde el orden es importante.

4. Etiquetas de Formularios

Los formularios permiten al usuario enviar información, como en un registro o contacto.

<form>, <input>, <label>, <select>, <textarea>

¿Qué son?

  • <form>: es el contenedor de todo el formulario.
  • <label>: es el texto que describe un campo.
  • <input>: es el campo donde el usuario escribe o selecciona.
  • <select>: es una lista desplegable de opciones.
  • <textarea>: es un área de texto para mensajes largos.

Sintaxis:

<form>
  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre">

  <label for="correo">Correo:</label>
  <input type="email" id="correo" name="correo">

  <label for="tema">Tema favorito:</label>
  <select id="tema" name="tema">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
  </select>

  <label for="mensaje">Mensaje:</label>
  <textarea id="mensaje" name="mensaje"></textarea>

  <button type="submit">Enviar</button>
</form>
Buenas prácticas: Siempre enlaza el <label> con su campo usando los atributos for e id con el mismo valor. Esto mejora la accesibilidad.

5. Etiquetas de Tablas

Las tablas sirven para mostrar información en filas y columnas, como una hoja de cálculo.

<table>, <tr>, <th>, <td>

¿Qué son?

  • <table>: crea la tabla.
  • <tr>: crea una fila (table row).
  • <th>: celda de encabezado (sale en negrita).
  • <td>: celda con datos normales.

Sintaxis:

<table>
  <tr>
    <th>Nombre</th>
    <th>Edad</th>
    <th>Ciudad</th>
  </tr>
  <tr>
    <td>Ana García</td>
    <td>20</td>
    <td>Bogotá</td>
  </tr>
  <tr>
    <td>Luis Pérez</td>
    <td>22</td>
    <td>Medellín</td>
  </tr>
</table>

Demostración:

Nombre Edad Ciudad
Ana García 20 Bogotá
Luis Pérez 22 Medellín
Buenas prácticas: Las tablas son para datos organizados en filas y columnas. No las uses para organizar el diseño de la página (para eso está CSS).

6. Etiquetas Multimedia

Estas etiquetas permiten agregar imágenes, audio y video a la página.

<img> — Imagen

¿Qué es? Muestra una imagen en la página. No tiene etiqueta de cierre. El atributo src es la ruta de la imagen y alt es el texto alternativo.

Sintaxis:

<img src="mi-foto.jpg" alt="Descripción de la imagen" width="300">

Demostración:

Buenas prácticas: El atributo alt es obligatorio. Describe la imagen para personas que usan lectores de pantalla o cuando la imagen no carga.

<video> — Video

¿Qué es? Incrusta un video directamente en la página sin necesidad de YouTube. El atributo controls muestra los controles de reproducción.

Sintaxis:

<video src="mi-video.mp4" controls width="400">
          

        

 Buenas prácticas: Siempre escribe un texto de respaldo entre las etiquetas por si el navegador no soporta el video.

<audio> — Audio

¿Qué es? Incrusta un reproductor de audio en la página.

Sintaxis:

<audio src="mi-cancion.mp3" controls>
  Tu navegador no puede reproducir este audio.
</audio>


Buenas prácticas: No uses autoplay sin muted. El audio automático molesta a los usuarios.
